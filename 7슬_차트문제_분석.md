# 7슬라이드 차트 문제 분석 (7슬스샷1203 기준)

## 1. 현상 (스크린샷에서 보이는 것)
- **왼쪽 차트**: 곡선(회색 2024)이 **흰색 차트 박스의 아래 경계 밖으로 삐져나옴**. 흰 박스는 어느 높이에서 끝나는데 곡선은 그 아래 파란 배경까지 계속 내려감 → "박스 마감이 안 된 것처럼" 보임.
- **오른쪽 결론(BUT) 박스**: 박스 바닥이 슬라이드 하단에 거의 붙어 깔끔한 여백 없이 끝남 → 역시 "안 닫힌 것처럼" 보임.
- 즉 핵심은 **세로 공간 부족(overflow)** + **차트 SVG가 자기 박스보다 커서 박스 밖으로 넘침**.

## 2. 근본 원인
1. **글씨를 전부 26px로 키우면서 7슬 전체 콘텐츠가 720px 프레임을 초과**함.
   - 슬라이드 가용 세로 ≈ 592px(상하 패딩 64 제외) − 헤더 ≈ 92 − distWrap 상단여백 24 = **약 476px**.
   - 오른쪽 열(과락 막대 박스 + 결론 박스 + 간격)이 26px 글씨로 **약 500px**까지 커져서 476px를 넘김 → 박스 바닥이 슬라이드 밖으로 밀려 잘림.
2. **차트 SVG 높이 처리 방식 문제**.
   - 현재 `#distChart`는 `height:auto`(가로 100%에 비례한 고정 높이). viewBox를 440으로 키워서 SVG 자체가 더 높아짐.
   - 이 고정 높이가 **흰 박스(차트 패널)의 실제 가용 높이보다 커서**, 패널 안에 안 들어가고 **아래로 넘쳐 흰 박스 밖에 곡선이 그려짐**. (패널에 overflow 클립도 없음)

## 3. 내가 그동안 한 것 / 왜 안 통했나 (솔직 정리)
- (1차) SVG에 `flex:1; min-height:0` 추가 → 패널 높이에 맞춰 줄어들게 한 시도. 그러나 그때는 **distWrap에 `flex:1`이 남아 있어 박스가 슬라이드 바닥까지 늘어나** "밑이 짤린" 것처럼 보였음.
- (2차) distWrap·SVG의 `flex:1`을 **둘 다 제거** → distWrap은 박스가 닫혔지만, **SVG가 다시 `height:auto`(고정 높이)로 돌아가 패널 밖으로 삐져나오는 지금 상태**가 됨.
- 결론: 두 개를 따로 본 게 실수. **distWrap은 강제높이 빼고(닫힘 유지) + SVG는 패널 안에 맞추기(flex)** 를 **동시에** 해야 함. 추가로 **오른쪽 열이 476px를 넘지 않도록 높이를 줄여야** 함.

## 4. 해결안 (제안)
1. **차트 SVG를 패널 안에 가두기**: `#distChart`에 `flex:1; min-height:0` + `preserveAspectRatio="xMidYMid meet"` → 패널 가용 높이에 맞춰 **비율 유지하며 축소**(잘림·넘침 모두 없음). `height:auto` 제거.
2. **distWrap은 강제 높이 채움(flex:1) 사용 안 함**(현재대로) → 박스가 내용 크기로 닫힘.
3. **오른쪽 열 높이 ≤ 476px로 축소** (현재 ~500px):
   - `.distNote` 패딩 22→16, 박스 간격 18→14
   - 과락 막대 SVG 높이 약간 축소
   - 결론 박스의 보조 캡션(`출제기조 전환·쉬워짐`, `고득점 경쟁 심화`)은 **26px 예외로 22px 정도 허용** 검토 (이 보조 라벨까지 26px면 7슬은 물리적으로 안 들어감)
4. 위 적용 후, 왼쪽 차트 박스와 오른쪽 결론 박스의 **바닥 모서리가 슬라이드 안쪽에서 깔끔히 닫히는지** 확인.

## 5. 확인 필요 (사용자 결정)
- 7슬 오른쪽 보조 캡션 2곳을 **26px 미만(예: 22px) 예외**로 허용해도 될까요?
  - 허용 → 깔끔하게 한 화면에 들어감.
  - 불가(26px 고수) → 결론/과락 박스 내용을 더 줄이거나 레이아웃을 2행으로 재구성해야 함.
